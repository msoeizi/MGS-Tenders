MGS PARAMETER REGISTRY
Action: initial_document_review
Version: 1.0 (Discovery Phase)
1. INPUT PARAMETERS
1.1 project_id

Type: string
Description:
Unique identifier for the project. Generated by the system at project creation.

1.2 documents[]

Type: array of objects

Each object:

document_id

Type: string
Description:
Unique identifier for each uploaded document.

document_title

Type: string
Description:
Original file name or title of the uploaded document.

2. OUTPUT PARAMETERS

The action returns a single structured object containing the following top-level sections:

project_info
project_contacts
millwork_schedule
finish_schedule
estimate_prefill
review_flags
evidence_index
3. PROJECT INFO
project_info
project_title

Type: string
Description:
Name of the project as inferred from documents.

project_address

Type: string
Description:
Full project address extracted from documents.

project_description

Type: string
Description:
Short descriptive summary of the project scope and nature.

project_type

Type: string
Description:
Classification of project type. Examples:

retail retrofit
medical new build
public school renovation
scope_classification

Type: string
Allowed values:

install_only
supply_and_install

Description:
Best estimate of whether MGS scope includes supply + install or install only.

Uncertainty must be handled via:

confidence
review_flags
evidence_index
distance_from_47_geraldton

Type: string (or number – to be finalized later)
Description:
Driving distance from:
47 Geraldton Crescent, North York, Ontario

This may later expand into:

distance_km
drive_time_minutes
4. PROJECT CONTACTS
project_contacts[]

Array of relevant contacts only.

Each object:

company

Type: string
Description:
Company name associated with the contact.

contact_name

Type: string
Description:
Person’s name.

email

Type: string
Description:
Email address if found.

phone_number

Type: string
Description:
Phone number if found.

category

Type: string
Description:
Role of the contact.

Examples:

architect
interior_designer
consultant
electrical
mechanical
structural
general_contractor
owner
other
5. MILLWORK SCHEDULE
millwork_schedule[]

Each entry represents a functional millwork item, not small components.

item_id

Type: string
Description:
Unique identifier for the item.

parent_item_id

Type: string or null
Description:
Used for nesting. Null = parent item.

item_name

Type: string
Description:
Name of the millwork item. Example:

Reception Desk
Washroom Millwork
Kitchen Millwork
room_area

Type: string
Description:
Room or area where the item is located.

tags

Type: array of strings
Description:
Categorization of item.

Examples:

cabinetry
countertop
wall_panel
slatwall
upholstery
glass
metalwork
finish_carpentry
scope_description

Type: string
Description:
GPT-generated SOW-style description of the item.

finish_codes

Type: array of strings
Description:
Links to finish_schedule entries.

confidence

Type: integer (percentage 0–100)
Description:
Confidence level of item detection and classification.

evidence_refs

Type: array of evidence_id
Description:
Links to supporting evidence.

6. FINISH SCHEDULE
finish_schedule[]

Each entry represents a material/finish used in the project.

finish_code

Type: string
Description:
Unique identifier (e.g., PL1, M2).

material_name

Type: string
Description:
Human-readable name of the material.

material_category

Type: string
Description:
Type of material.

Examples:

laminate
melamine
veneer
quartz
solid_surface
metal
glass
unit_type

Type: string
Description:
Measurement unit.

Examples:

sheet
sqft
linear_ft
slab
unit_cost

Type: number
Description:
Cost per unit.

notes

Type: string
Description:
Additional details or assumptions.

7. ESTIMATE PREFILL
estimate_prefill[]

Each entry corresponds to a millwork item.

item_id

Type: string
Description:
Reference to millwork_schedule.item_id

material_breakdown[]

Array of material usage entries.

finish_code

Type: string
Description:
Reference to finish_schedule.finish_code

quantity

Type: number
Description:
Estimated quantity of material used.

basis_note

Type: string
Description:
Explanation of how quantity was estimated.

hardware_cost

Type: number
Description:
Single placeholder value for hardware cost.

design_hours

Type: number
Description:
Estimated design time.

fabrication_headcount

Type: number
Description:
Number of workers required for fabrication.

fabrication_hours_each

Type: number
Description:
Hours per worker for fabrication.

install_headcount

Type: number
Description:
Number of installers.

install_hours_each

Type: number
Description:
Hours per installer.

misc_cost_placeholder

Type: number
Description:
Additional cost for miscellaneous items.

subcontractor_flag

Type: boolean
Description:
Indicates if subcontractor costing is required.

Triggers:

stone
quartz
solid surface
postformed countertop
metalwork
glass / mirror

flag_id

related_item_id

Type: string or null
Description:
Null means project-level flag.

category

Type: string
Description:
Type of issue.

Examples:

severity

Type: string
Description:
What the user should do.

evidence_refs

Type: array of evidence_id
Description:
Links to supporting evidence.

GPT Action schema
